Important FAQ

What is the Document Object Model (DOM)?

The DOM (Document Object Model) is a programming interface for web documents. It represents the structure of a web page in a hierarchical tree format, allowing developers to access, manipulate, and modify the content, structure, and style of the webpage dynamically using programming languages like JavaScript.

If you want to grab all elements with a specific class, use document.getElementsByClassName()

What is the delegate in the context of the DOM,and why it is useful?

Performance Improvement: Instead of attaching individual event listeners to multiple child elements, you attach just one to a parent element. This reduces the number of event listeners and improves performance, especially in cases where there are many child elements (e.g., a long list). Handles Dynamic Elements: Event delegation is particularly useful when dealing with elements added dynamically (i.e., after the page has loaded). Since the parent element's listener captures events even for new child elements, you don't need to add new listeners when elements are dynamically inserted. Simpler Code: Maintaining just one event listener on a parent element is simpler and cleaner compared to adding many listeners to individual child elements.
